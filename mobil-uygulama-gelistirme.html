Mobil Uygulama Geliştirme

Android ADB Development

Dersin Konusu

Bu derste mobil uygulama geliştirme hakkında temel kavramlar, kullanılan araçlar, frameworkler ve örnek uygulamalar ele alınacaktır.

A. ADB Sunucu Yönetimi

adb kill-server

- ADB sunucusunu sonlandırır.

- Arka planda çalışan ADB işlemlerini durdurur.

adb start-server

- ADB sunucusunu başlatır.

- Cihazlarla iletişimi yeniden başlatır.

adb root

- ADB’yi root (yönetici) yetkisiyle yeniden başlatır.

- Bazı komutların çalışması için gerekebilir.

B. ADB Yeniden Başlatma Komutları

adb reboot

- Cihazı normal şekilde yeniden başlatır.

adb reboot recovery

- Cihazı recovery (kurtarma) modunda yeniden başlatır.

adb reboot-bootloader

- Cihazı bootloader (önyükleyici) moduna alır.

- Genellikle ROM flash işlemleri için kullanılır.

C. ADB Shell ve Cihaz Üzerinde Komut Çalıştırma

adb shell

- Cihaz üzerinde bir terminal oturumu açar.

- Doğrudan cihazın komut satırında komutlar çalıştırabilirsiniz.

D. Cihaz Bağlantısı ve Bilgi Alma

adb usb

- USB üzerinden bağlantı kurmayı sağlar.

adb devices

- Bağlı cihazların listesini görüntüler.

- adb devices -l ile cihazların detaylı bilgileri de listelenir.

adb connect [ip_adresi]

- Belirtilen IP adresine sahip cihazla kablosuz bağlantı kurar.

adb shell getprop ro.build.version.release

- Cihazın yüklü olan Android sürümünü getirir.

E. Logcat ve Hata Raporlama

adb logcat

- Cihazın log çıktısını gerçek zamanlı gösterir.

- Hata ayıklama sırasında logları takip etmek için kullanılır.

adb logcat -c

- Cihazdaki mevcut logları temizler.

- Temiz bir log dosyası ile çalışmak için yararlıdır.

adb logcat -d > [dosya_yolu]

- Logcat çıktısını belirtilen dosyaya kaydeder.

adb bugreport > [dosya_yolu]

- Cihazın tüm sistem bilgilerini, loglarını, dumpsys ve dumpstate çıktısını içeren raporu dosyaya yazar.

F. Dosya Transferi Komutları

adb push [kaynak] [hedef]

- Bilgisayardan cihaza dosya kopyalamak için kullanılır.

adb pull [cihaz_dosya_yolu] [yerel_yol]

- Cihazdaki dosyayı bilgisayara kopyalar.

G. Uygulama Kurma ve Kaldırma

adb -e install path/to/app.apk

- Emülatöre uygulama yükler.


Uygulama kurulumu için ek bayraklar:

  • -d: Sadece USB bağlı cihaza yönlendirir.
  • -e: Sadece emülatöre yönlendirir.
  • -s <seri numarası>: Belirli bir cihaza yükleme yapar.
  • -p <ürün adı veya yol>: Ürüne veya cihaza özel seçenek verir. 
adb devices | tail -n +2 | cut -sf 1 | xargs -IX adb -s X install -r com.myAppPackage

- Bu komut, bağlı tüm cihazlara uygulamayı yeniden kurar (-r: verileri korur).

adb uninstall com.myAppPackage

- Belirtilen paket adındaki uygulamayı kaldırır.

adb uninstall -k <apk_adı>

- Uygulamayı kaldırırken verileri saklar.

adb shell pm uninstall com.example.MyApp

- Alternatif olarak cihaz üzerinden uygulamayı kaldırır.

adb shell pm clear [paket_adı]

- Uygulamanın tüm verilerini temizler.

adb devices | tail -n +2 | cut -sf 1 | xargs -IX adb -s X uninstall com.myAppPackage

- Toplu kaldırma örneği.

adb install -r yourApp.apk

- Uygulamayı verileri koruyarak yeniden kurar.

adb install –k <apk_yolu>

- Alternatif güncelleme yöntemi; uygulama verilerini saklar.

H. Aktivite Yönetimi ve Intent İşlemleri

adb shell am start -W -c android.intent.category.HOME -a android.intent.action.MAIN

- Cihazın ana ekranını (launcher) başlatır.

adb shell am start -a android.intent.action.VIEW

- Belirtilen aksiyonla bir aktivite başlatır (örneğin, URL açma).

adb shell am broadcast -a 'my_action'

- Özel bir aksiyonu tüm uygulamalara yayınlar.

adb shell am start -a android.intent.action.CALL -d tel:+905551234567

- Belirtilen numaraya telefon çağrısı başlatır.

adb shell am start -a android.intent.action.SENDTO -d sms:+972527300294 --es sms_body "Test --ez exit_on_sent false"

- SMS gönderme ekranını açar ve mesajı önceden doldurur.

I. İzin Yönetimi ve Cihaz Emülasyonu

adb shell pm reset-permissions -p your.app.package

- Uygulamanın mevcut izinlerini sıfırlar.

adb shell pm grant [paket_adı] [izin]

- Belirtilen uygulamaya izin verir.

adb shell pm revoke [paket_adı] [izin]

- Uygulamadan izni geri alır.

adb shell wm size 2048x1536

- Cihazın ekran çözünürlüğünü değiştirir.

adb shell wm density 288

- Ekran yoğunluğunu (dpi) ayarlar.

adb shell wm size reset

- Varsayılan ekran ayarlarına döner.

adb shell wm density reset

- Varsayılan ekran yoğunluğuna döner.

J. Metin Girişi ve Klavye Olayları

adb shell input text 'Mesajınız'

- Cihaza metin girdisi gönderir.

adb shell input keyevent 3

- Ana ekran (home) butonuna basar.

adb shell input keyevent 4

- Geri butonuna basar.

adb shell input keyevent 66

- Enter tuşuna basar.

adb shell input keyevent 67

- Silme (backspace) işlemi yapar.

adb shell input keyevent 26

- Güç düğmesi (açma/kapama)

adb shell input keyevent 27

- Kamera tuşu

K. Ekran Görüntüsü Alma ve Video Kaydı

adb shell screencap -p /sdcard/screenshot.png

- Cihazın ekran görüntüsünü belirtilen dosya yoluna kaydeder.

adb pull /sdcard/screenshot.png

- Alınan ekran görüntüsünü bilgisayarınıza kopyalar.

adb.exe exec-out screencap -p > screen.jpg

- Komut satırından direkt olarak görüntüyü bir dosyaya aktarır.

adb shell screenrecord /sdcard/demo.mp4

- Cihazın ekranını video olarak kaydeder.

- Kaydı durdurmak için Ctrl+C kullanılır.

adb shell "am start -a android.media.action.VIDEO_CAPTURE"

- Video kaydı başlatır.

L. Yedekleme, Geri Yükleme ve Sideload İşlemleri

adb backup -apk -all -f backup.ab

- Tüm uygulamaları ve ayarları yedekler.

adb backup -apk -shared -all -f backup.ab

- Uygulama verileri ve paylaşılan dosyaları yedekler.

adb backup -apk -nosystem -all -f backup.ab

- Sadece sistem dışı (kullanıcı) uygulamaları yedekler.

adb restore backup.ab

- Daha önce alınmış yedeği cihaza geri yükler.

adb sideload

- Bilgisayarınızdan cihaza özel ROM veya zip dosyalarını yüklemek için kullanılır.

M. Fastboot Komutları

fastboot devices

- Bootloader modunda bağlı cihazları listeler.

- ADB'nin bootloader modundaki eşdeğeridir.

N. Shared Preferences İşlemleri

adb shell 'am broadcast -a org.example.app.sp.PUT --es key key_name --es value "hello world!"'

- Uygulamanın varsayılan shared preferences dosyasına "hello world!" değeri ekler. 

adb shell 'am broadcast -a org.example.app.sp.REMOVE --es key key_name'

- Belirtilen anahtarla kayıtlı değeri siler.

adb shell 'am broadcast -a org.example.app.sp.CLEAR --es key key_name'

- Shared preferences içeriğini temizler.

adb shell 'am broadcast -a org.example.app.sp.CLEAR --ez restart true'

- Değişikliklerin aktif olması için uygulamayı yeniden başlatır.

O. Monkey Test Aracı

adb shell monkey -p com.myAppPackage -v 10000 -s 100

- Belirtilen uygulama üzerinde rastgele 10.000 olay (dokunma, kaydırma vb.) üretir.

- Uygulamanın stres testi ve hata tespiti amacıyla kullanılır.

P. Bash Komutları ve Alias Oluşturma

alias apkinstall="adb devices | tail -n +2 | cut -sf 1 | xargs -I X adb -s X install -r \$1"

- Uygulama Kurulum Alias'ı

- Bağlı tüm cihazlara belirtilen APK'yı kurar.

alias rmapp="adb devices | tail -n +2 | cut -sf 1 | xargs -I X adb -s X uninstall \$1"

- Uygulama Kaldırma Alias'ı

- Bağlı tüm cihazlardan belirtilen uygulamayı kaldırır.

alias clearapp="adb devices | tail -n +2 | cut -sf 1 | xargs -I X adb -s X shell pm clear \$1"

- Uygulama Verilerini Temizleme Alias'ı

- Bağlı tüm cihazlarda uygulamanın verilerini temizler.

alias startintent="adb devices | tail -n +2 | cut -sf 1 | xargs -I X adb -s X shell am start \$1"

- Intent Başlatma Alias'ı

- Bağlı tüm cihazlarda belirtilen intent'i çalıştırır.

B. Framework ve Çapraz Platform Uygulama Geliştirme

Framework Nedir?

Uygulama geliştirmeyi kolaylaştıran, yapı ve kütüphanelerin toplandığı altyapıdır.

Popüler Çapraz Platform Frameworkleri:

Flutter:

  • Dil: Dart
  • Örnek Uygulamalar: eBay, Alibaba, Google Pay, ByteDance

React Native:

  • Dil: JavaScript
  • Örnek Uygulamalar: Microsoft Office, Skype, Xbox Game Pass, Facebook, Desktop Messenger, Oculus

Kotlin Multiplatform:

  • Dil: Kotlin
  • Örnek Uygulamalar: McDonald's, Netflix, Forbes, 9GAG, Cash App, Philips

Ionic:

  • Dil: JavaScript
  • Örnek Uygulamalar: T-Mobile, BBC (Çocuk ve Eğitim Uygulamaları), EA Games

.Net MAUI:

  • Diller: C#, XAML
  • Örnek Uygulamalar: NBC Sports Next, Escola Agil, Irth Solutions

NativeScript:

  • Diller: JavaScript, TypeScript
  • Örnek Uygulamalar: Daily Nanny, Strudel, Breethe

C. Mobil İşletim Sistemleri

Android:
  • Geliştirici: Google
  • Kullanılan Cihazlar: Telefon, Tablet, Wearable, TV, Otomotiv, PC
  • Özellikler: Açık kaynaklı, geniş cihaz yelpazesi, özelleştirilebilir arayüz
  • Programlama Dilleri: Dart (Flutter), Kotlin, Java, C++, C# (MAUI/Xamarin), Python
  • Kullanılan IDE'ler: Android Studio, ayrıca VSCode, Visual Studio, RAD Studio, IntelliJ IDEA gibi çapraz platform araçları.
iOS:
  • Geliştirici: Apple Inc.
  • Kullanılan Cihazlar: iPhone, iPad, iPod Touch
  • Özellikler: Güvenli, kullanıcı dostu arayüz, Apple ekosistemine sıkı entegrasyon
  • Programlama Dilleri: Dart (Flutter), Swift, Objective-C, C# (MAUI/Xamarin)
  • Ana IDE: Xcode (çapraz platform geliştirme seçenekleri de mevcut).
